## Используемые методы:

### 1. DuckDuckGo Image Search

- **Файл:** `methods/duckduckgo.py`
- **Библиотека:** `duckduckgo_search.DDGS`
- **Описание:** Официальный текстовый поиск изображений DuckDuckGo через `ddgs.images()`.
- **URL источника:** `https://duckduckgo.com`
- **Особенности:**
  - Не требует API-ключа.
  - Возвращает заголовок, прямую ссылку на изображение, источник и thumbnail.
  - Может возвращать пустой результат при частых запросах.

### 2. DuckDuckGo Text Search

- **Файл:** `methods/duckduckgo.py`
- **Библиотека:** `duckduckgo_search.DDGS`
- **Описание:** Текстовый поиск DuckDuckGo с фильтром `image`.
- **URL источника:** `https://duckduckgo.com`
- **Особенности:**
  - Возвращает обычные веб-ссылки и сниппеты, а не прямые URL изображений.
  - Полезен для поиска страниц с изображениями.

### 3. Bing Image Downloader

- **Файл:** `methods/bing.py`
- **Библиотека:** `bing_image_downloader`
- **Описание:** Скачивает изображения из Bing по запросу локально.
- **URL источника:** `https://www.bing.com/images/search`
- **Особенности:**
  - Создаёт поддиректорию `search_results/bing_downloader/<query>`.
  - Результат — список путей к скачанным файлам.
  - Может столкнуться с антибот-защитой при массовом использовании.

### 4. iCrawler Google Images

- **Файл:** `methods/icrawler_google.py`
- **Библиотека:** `icrawler.builtin.GoogleImageCrawler`
- **Описание:** Краулер Google Images.
- **URL источника:** `https://www.google.com/search?tbm=isch`
- **Особенности:**
  - Сохраняет файлы в `search_results/icrawler_google/<query>`.
  - Google часто блокирует автоматические запросы, требуя обход защиты.
  - Рекомендуется использовать с задержками и прокси.

### 5. iCrawler Bing Images

- **Файл:** `methods/icrawler_bing.py`
- **Библиотека:** `icrawler.builtin.BingImageCrawler`
- **Описание:** Краулер Bing Images.
- **URL источника:** `https://www.bing.com/images/search`
- **Особенности:**
  - Сохраняет файлы в `search_results/icrawler_bing/<query>`.
  - Работает стабильнее, чем Google-краулер.
  - При массовых запросах возможна блокировка IP.

### 6. Wikipedia Scraping

- **Файл:** `methods/wikipedia.py`
- **Библиотеки:** `requests`, `beautifulsoup4`
- **Описание:** Парсинг картинок со страницы английской Википедии по первому слову запроса.
- **URL источника:** `https://en.wikipedia.org/wiki/<term>`
- **Особенности:**
  - Использует только первое слово запроса.
  - Преобразует относительные URL в абсолютные.
  - Возвращает URL, alt и title.

### 7. Flickr Search

- **Файл:** `methods/flickr.py`
- **Библиотеки:** `requests`, `lxml`
- **Описание:** Парсинг поисковой выдачи Flickr.
- **URL источника:** `https://www.flickr.com/search/?text=<query>`
- **Особенности:**
  - Возвращает URL и alt-текст найденных изображений.
  - Структура HTML Flickr может измениться, что сломает парсер.
  - Изображения могут быть защищены лицензией.

### 8. Wikimedia Commons

- **Файл:** `methods/wikimedia_commons.py`
- **Библиотеки:** `requests`, `beautifulsoup4` + `html5lib`
- **Описание:** Парсинг категории Wikimedia Commons.
- **URL источника:** `https://commons.wikimedia.org/wiki/Category:<query>`
- **Особенности:**
  - Большинство изображений находятся в общественном достоянии или под свободными лицензиями.
  - Пробелы в запросе заменяются на `_`.
  - Может вернуть 404, если категория не существует.

### 9. Lorem Picsum

- **Файл:** `methods/lorem_picsum.py`
- **Библиотеки:** `requests`
- **Описание:** Получение случайных изображений с Lorem Picsum.
- **URL источника:** `https://picsum.photos/v2/list`
- **Особенности:**
  - Идеально для тестов и прототипов.
  - Параметр `query` не влияет на результат.
  - Возвращает id, автор, URL и ссылку на оригинал.

### 10. Placeholder APIs

- **Файл:** `methods/placeholder.py`
- **Библиотеки:** `requests`
- **Описание:** Проверка доступности placeholder-сервисов.
- **URL источника:**
  - `https://via.placeholder.com/800x600?text=<query>`
  - `https://placehold.co/800x600?text=<query>`
  - `https://placekitten.com/800/600`
- **Особенности:**
  - Выполняет HEAD-запросы, не скачивает изображения.
  - Возвращает статус и флаг доступности.

### 11. ArtStation API

- **Файл:** `methods/artstation.py`
- **Библиотеки:** `requests`
- **Описание:** Публичный JSON endpoint последних работ ArtStation.
- **URL источника:** `https://www.artstation.com/projects.json`
- **Особенности:**
  - Параметр `query` игнорируется API.
  - Возвращает обложки (small square), заголовок и имя художника.
  - Все работы защищены авторским правом, требуется лицензия для использования.

### 12. NASA Image API

- **Файл:** `methods/nasa.py`
- **Библиотеки:** `requests`
- **Описание:** Поиск изображений в библиотеке NASA Image and Video Library.
- **URL источника:** `https://images-api.nasa.gov/search`
- **Особенности:**
  - Публичный API, не требует ключа.
  - Большинство материалов находятся в общественном достоянии, но требуется указание источника.
  - Возвращает заголовок, URL и описание.

### 13. Met Museum API

- **Файл:** `methods/met_museum.py`
- **Библиотеки:** `requests`
- **Описание:** Поиск объектов из Metropolitan Museum of Art.
- **URL источника:**
  - `https://collectionapi.metmuseum.org/public/collection/v1/search`
  - `https://collectionapi.metmuseum.org/public/collection/v1/objects/<id>`
- **Особенности:**
  - Публичный API.
  - Многие изображения находятся в Public Domain/CC0.
  - Делает два запроса на объект: поиск + детали.

### 14. PIL Image Analysis

- **Файл:** `methods/pil_analysis.py`
- **Библиотека:** `Pillow`
- **Описание:** Анализ локального изображения.
- **Особенности:**
  - Если изображения нет, генерирует тестовый файл.
  - Возвращает формат, режим, размеры.
  - Не выполняет поиск в интернете.

### 15. EXIF Metadata Reading

- **Файл:** `methods/exif.py`
- **Библиотека:** `exifread`
- **Описание:** Чтение EXIF-метаданных локального изображения.
- **Особенности:**
  - Лучше всего работает с JPEG.
  - Генерирует тестовый файл при отсутствии.
  - Может содержать персональные данные (GPS, устройство, дата съёмки).

### 16. ImageIO Operations

- **Файл:** `methods/imageio_ops.py`
- **Библиотека:** `imageio`
- **Описание:** Чтение изображения и получение shape/dtype.
- **Особенности:**
  - Работает с локальными файлами.
  - Генерирует тестовый PNG при необходимости.
  - Не выполняет сетевых запросов.

### 17. OpenCV Detection

- **Файл:** `methods/opencv.py`
- **Библиотека:** `opencv-python-headless`
- **Описание:** Canny-детекция краёв на локальном изображении.
- **Особенности:**
  - Генерирует тестовый файл, если изображение отсутствует.
  - Возвращает форму исходного изображения и количество обнаруженных краёв.

### 18. Python-magic File Type

- **Файл:** `methods/python_magic.py`
- **Библиотека:** `python-magic`
- **Описание:** Определение MIME-типа локального файла по содержимому.
- **Особенности:**
  - Требует установки системной библиотеки `libmagic`.
  - Возвращает MIME-тип и флаг `is_image`.
  - Работает только с локальными файлами.

### 19. AIOHTTP URL Validator

- **Файл:** `methods/url_validator.py`
- **Библиотека:** `aiohttp`
- **Описание:** Асинхронная проверка доступности URL.
- **URL источника:** `https://picsum.photos/seed/<query>-<i>/800/600`
- **Особенности:**
  - Проверяет URL методом HEAD.
  - Возвращает статус и Content-Type.
  - Используется для валидации, а не для поиска.

### 20. Wikimedia Commons API

- **Файл:** `methods/wikimedia_api.py`
- **Библиотека:** `requests`
- **Описание:** Поиск изображений в Wikimedia Commons через MediaWiki API.
- **URL источника:** `https://commons.wikimedia.org/w/api.php`
- **Особенности:**
  - Использует action=query, generator=search, namespace=6 (файлы).
  - Возвращает URL файла, название и pageid.
  - Изображения обычно свободно лицензированы.
